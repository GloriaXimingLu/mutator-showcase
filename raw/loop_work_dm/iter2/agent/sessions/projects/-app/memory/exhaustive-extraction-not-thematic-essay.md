**What I changed (v0, this iteration):** Added a new §2 to `harness/skill/initial_skill.md` ("Extract every distinct issue across the WHOLE document set — do not write a thematic essay around a few chosen headline issues") and a §2 completeness self-check; renumbered the framework-audit and format sections to §3/§4. The rule: read every document given (including supporting ones — policies, records, valuations, appendices, letters), cross-reference each against every other (every mismatch = its own numbered finding), pull exact figures/names/dates/identifiers verbatim instead of generalizing them, give every distinct issue its own numbered finding rather than folding several into a theme, and explicitly dispose of distractor non-issues. Self-check: list every document; any that yielded zero findings was skipped or merged away — go back and mine it.

**Failure mode targeted (from my analysis of `rollouts/`):** Prior iterations fixed *format* (deliverable-as-prose, output-file-is-deliverable) — confirmed working: all current deliverables are full prose in the output file. The remaining failures are *substantive coverage*. Environmental ESA 0.17 (44/53 fail) — agent read the ESA + 6 supporting docs (transcript confirms) which contain the rubric entities (Tank 3, Triton Metals, Central Valley Chemical, NFA covers 2.3/14.7, PCB transformer, western yard, water well, floor drain), but the 261-line memo had 0 hits for all of them; it wrote deep prose on its own chosen themes (PCE plume, UST, data gaps, lender, indemnity) and dropped ~12 named issues. H-1B 0.67 — produced 15 issues (more than the rubric's ~11) but re-derived its own numbering, missing the rubric's immigration-status-gap, alternative-qualification-language, SOC-code, and TensorFlow/PyTorch/SageMaker issues, and wrongly flagged an OPT distractor. Employment 0.58 — strong 18-issue memo but missed a cluster that lives only in supporting docs (ORS 656.236, $25K liquidated damages and 12-month cap from the company policy, 409A savings clause, tax indemnification, IP-assignment confirmation). Trusts 0.72 — failures were exact figures not pulled from records ($123,300, $23,400, $6,200, $619,900, $981,435, etc.), generalized away. Cyber 0.73 — missed IRT-composition, CIRP-materiality-framework, 10-K Item 1C, CISO bottleneck, legal-hold, "file 8-K immediately", CIRP-predates-rules. The common thread: mines the primary doc, writes a thematic essay, silently drops granular findings — especially those requiring cross-referencing supporting docs and extracting exact figures.

**Why it generalizes:** method, not memorization — "enumerate every distinct issue across every document; cross-reference; pull specifics verbatim; any doc with zero findings was skipped" applies to any issue/memo/markup/findings task regardless of practice area and reads no task-specific facts, so it cannot force irrelevant artifacts (it tells the agent to extract what the record warrants, not to emit a fixed row). This is the next layer after [[emit-deliverable-not-work-summary]] and [[deliverable-file-not-cover-note]]: those fixed *whether substance reaches the page*; this fixes *how much of the record's substance reaches the page*.
