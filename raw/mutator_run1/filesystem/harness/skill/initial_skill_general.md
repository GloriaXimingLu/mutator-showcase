## How to use this playbook
Your measured weakness is coverage completeness, not depth — and you tend to **satisfy a rule's letter
without producing its output** (writing "Consequence:" with no consequence; naming "a" standard, not the
controlling one). So **every step must produce VISIBLE output in the deliverable. A step with no emitted
artifact has not been done.**

## 0. Frame the task and open every source
- **Restate the literal question** the task asks as the deliverable's first section, and answer THAT (a
  "reasonableness" or "assess" task needs a verdict, not only line-item compliance).
- **Open EVERY source — emit a SOURCES-OPENED ledger** (one row per document the instructions / exhibits /
  records name): **opened?** / **key constraints extracted** (the figures, mandated clause set, approved
  ranges, prohibited provisions). A referenced source with no row = task not started; if a named source is
  not in your files, say so in a row and convert it to a deficiency finding.

## 1. Extract, then close every fact (emit each labeled line under the finding)
- **Consequence:** the specific legal/financial result or required action — not a restatement of the fact.
- **Other theories:** route the fact to EVERY doctrine it implicates — including the adverse/counterparty
  reading AND the symmetric case where your own client later crosses the threshold. One-fact-one-frame fails.
- **Figure — COMPUTE, never copy or defer:** if a value has inputs/offset/base, do the arithmetic and print
  your labeled result (equity = value − lien; % = part ÷ base; excess = actual − cap × rate) — this covers
  non-dollar deltas too (uncovered area, spans, counts, durations). Compute each line item separately, then
  **aggregate related items to a category total and apply the controlling legal frame to that aggregate**,
  not only to the headline number. No computable may be parked "open / TBD."
- **Authority:** the specific controlling statute / regulation / section / clause AND the right BODY of law
  (the governing institution's rules where they override default procedure; the on-point regulation for the
  industry). A generic gesture at "the law," or a cite to the wrong library, is a miss.
- **Severity:** critical / high / medium / low by anchor — a direct financial interest is presumptively
  HIGH; an already-filed complaint / pending audit / active enforcement matter is top severity, carved out
  as a standalone immediate action separate from systemic remediation.

## 2. Hunt for what is ABSENT — emit an OMISSIONS LEDGER (your #1 failure, 38%)
Run the standard component set for your deliverable type (below) and write a VISIBLE table — one row per
item: **present / MISSING → drafted-or-flagged here / n/a**. A component with no row is a failure. When
reviewing a document, critique **the document itself** ("it fails to DEFINE / CITE / CLASSIFY X; it is
internally inconsistent on Y"), not only the underlying conduct. **Then run a symmetric SCOPE-AUDIT** — a
markup that only ADDS clauses and never strikes/scopes one is incomplete: for each operative clause, force a
verdict — **improperly-INCLUDES? / improperly-OMITS-a-carve-out? / policy-PROHIBITS-this?** — and emit
`STRIKE: <clause>` / `CAP TO: <limit>` / `ADD carve-out: <language>` for each.

## 3. Reconcile — visibly, even when you resolve it
For every fact appearing in more than one source — **including characterizations/labels, not just
numbers/dates** — emit a line: "Source A says X; Source B says Y; controlling = Z because ___." Silently
picking the right value is a miss; the discrepancy is the gradable finding. Independently RECOMPUTE every
stated fee/price/figure from its rule-base and flag mismatches. Test the deliverable's key dates/mechanics
against the CURRENT governing rule even if no source raises it. Confirm your OWN deliverable obeys any hard
cap. **One-sided audit (no source B needed):** for each material DISCLOSURE sentence, DEFINITION, or
ROLE/REPORTING-LINE statement, test it against the operative record and emit `<statement>; underlying fact =
<X>; verdict = ACCURATE / MISLEADING / INSUFFICIENT / STRUCTURALLY-DEFICIENT because ___`.

## 4. Verdicts, synthesis, recommendation
- **Per-cell VERDICT:** for each (requirement × jurisdiction/item) cell, cite the specific threshold and
  state COMPLIANT or VIOLATION with the entity's value beside the required value. Never call a cell compliant
  by assuming a uniform policy satisfies a jurisdiction — check that jurisdiction's actual rule. A directional
  error (calling a violation "compliant") is high-severity.
- **Cross-issue synthesis:** add a section naming the MATERIAL compounding pair(s) and the holistic legal
  conclusion WITH its doctrine — distinct from the per-issue severities.
- **Recommendation:** end with the canonical next step for the type, and cost it against any stated budget.

## 5. Assert nothing false (commission guard)
- **Missing-fact protocol.** If an analysis needs a fact / party / member / figure NOT stated in a provided
  document, you may NOT supply, infer, or invent it. Write **`[NOT IN RECORD]`** and convert the gap into a
  finding ("the papers do not plead all members; the analysis cannot be completed and the filing is
  deficient"). Never name an entity not in the documents, and never put a real person in a role the documents
  never give them.
- **Resolve every value and all scaffolding.** A delivered draft STATES every value/price/date the
  instructions fix — no `[___]`/`[Party]` placeholders AND no prose-deferral ("assumed / illustrative only /
  TBD"). Populate every required table with real source figures.
- **Cross-deliverable consistency.** Multiple deliverables must agree — a certificate may not certify a
  condition your own memo flags as pending; a party's role stays consistent throughout.

## Deliverable-type component sets (drive §2's ledger — GENERIC; fill specifics from the actual record)
- **Drafting / markup:** standard clauses present? (return-of-property, IP assignment, governing-law,
  forum-selection, confidentiality (mutual?), liquidated-damages, deferred-comp/tax compliance,
  indemnification — who bears reclassification/characterization risk?, cooperation). When one attribute of a
  clause is defective, audit ALL of them (duration, scope, geography, consideration, triggers, exceptions,
  response windows). Map each source condition / CP / deadline to the clause implementing it, with value+basis.
- **Issue / analysis memo:** per-issue (consequence + authority + severity); a **non-issues / cleared-items**
  section; a **cross-issue + overall-readiness** conclusion distinct from the per-issue severities.
- **Financial / invoice / asset review:** a consolidated schedule (one row per item with its COMPUTED net) +
  an estate/portfolio TOTAL; the rule-mandated basis per number; **benchmark any owner-employee figure
  before calling it reasonable**; aggregate related items to a category total and apply the controlling
  frame; sweep every line for every violation type **symmetrically** (senior doing junior work AND junior
  overcharge; over-inclusion AND under-inclusion).
- **Requirements / compliance extraction:** sweep the governing CODE/ordinance itself (not the applicant's
  submission checklist), one row per obligation with its **phase** and **non-compliance consequence**; then
  sweep every NON-code source (financing/program agreements, bonds, covenants, MOUs) for ongoing duties; drop
  to the per-OWNER level for any owner-level threshold rule; compute any partial first reporting period.
- **Diligence / document-vs-document:** for each section walk a fixed checklist (depth, counts, standards,
  cited authorities, deadlines, survival under the deal structure) and state what the reviewed document
  FAILED to do, citing the governing standard/section.
- **Contract negotiation:** every contested term lands STRICTLY between both parties' poles (a value equal to
  a pole — uncapped, royalty-free, a fixed multiple replacing a requested range — is an automatic FAIL; if a
  red line forces a pole, add an offsetting protective mechanism, e.g. floor + put). Retain EVERY proposed
  mechanism in recognizable form; honor named, document-specific carve-outs rather than generic catch-alls.

## Self-review (blocks on the emitted artifacts)
Confirm, in VISIBLE text: the SOURCES-OPENED ledger is present (every named doc has a row); every fact
carries Consequence + Other-theories + (computed) Figure + Authority + Severity; the omissions-ledger table
is present with every MISSING item drafted AND the symmetric scope-audit ran; reconciliations — incl.
characterizations AND the one-sided audit of disclosures/definitions/reporting-lines — are printed even where
resolved; per-cell compliance VERDICTs stated; the cross-issue synthesis + recommendation present; no
invented `[NOT IN RECORD]` value and no placeholder/prose-deferral remains; multi-deliverables agree.
Format validation is NOT a completeness check.
